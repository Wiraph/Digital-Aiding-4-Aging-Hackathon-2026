# โค้ชฟื้นฟูการเอื้อมแขน

ต้นแบบระบบคัดกรองการเอื้อมแขนที่บ้านสำหรับงาน Digital Aiding 4 Aging Hackathon 2026 ระบบช่วยให้ผู้สูงอายุทำแบบทดสอบ 9 ช่องผ่านกล้อง และสร้างแดชบอร์ดสำหรับแพทย์หรือนักกายภาพบำบัดเพื่อดูการใช้แขนซ้าย-ขวา คุณภาพการเคลื่อนไหว และความเสี่ยง Learned Non-Use เบื้องต้น

ระบบนี้เป็นเครื่องมือช่วยคัดกรองและติดตามผล ไม่ใช่ระบบวินิจฉัยทางการแพทย์

## สิ่งที่มีในเวอร์ชันปัจจุบัน

- แอป React + Vite + Tailwind
- หน้าขอความยินยอมก่อนใช้กล้องและก่อนบันทึกวิดีโอ
- โปรไฟล์ผู้รับการทดสอบแบบไม่ระบุตัวตน
- หน้าปรับเทียบกล้องโดยตรวจการมองเห็นท่าทางและการยกมือทั้งสองข้าง
- แบบทดสอบเอื้อมแขน 9 ช่อง จำนวน 60 เป้าหมาย
- MediaPipe PoseLandmarker สำหรับตรวจใบหน้า ไหล่ ศอก ข้อมือ และสะโพก
- Overlay โครงร่างร่างกาย จุดใบหน้า วงมือซ้าย/ขวา และป้ายภาษาไทย
- หน้าทดสอบแบบ camera-first พร้อมกล่องเป้าหมาย ปุ่มควบคุมลอย และกดหน้ากล้องเพื่อเต็มจอ
- โหมดสาธิตสำรองเมื่อกล้องหรือโมเดลไม่พร้อม
- คะแนนรายครั้งด้านความเร็ว ความแม่นยำ ความลื่นไหล เส้นทางการเคลื่อนไหว และคุณภาพ landmark
- หน้าสรุปผลผู้รับการทดสอบแบบเข้าใจง่าย
- แดชบอร์ดแพทย์/นักกายภาพพร้อมเปรียบเทียบซ้าย-ขวา heatmap รายงานอัตโนมัติ ส่งออก JSON และลบ session
- เก็บข้อมูลโปรไฟล์และผลสรุปไว้ใน `localStorage`

## แนวคิดทางคลินิก

MVP ใช้ pose landmarks เป็นสัญญาณหลัก เพราะการติดตามข้อมือจากกล้องระยะไกลมีความเสถียรกว่าการติดตามนิ้วมือหรือฝ่ามือ

landmark ที่ใช้:

- ใบหน้า
- ไหล่ซ้าย/ขวา
- ศอกซ้าย/ขวา
- ข้อมือซ้าย/ขวา
- สะโพกซ้าย/ขวา

ระดับผลลัพธ์:

- ดี
- ควรติดตาม
- ควรฝึกเพิ่มเติม

ดัชนี Learned Non-Use Risk เป็นคะแนนต้นแบบที่รวมความไม่สมดุลของคะแนนแขนซ้าย-ขวา คะแนนของข้างที่อ่อนกว่า และช่องว่างของสัดส่วนการใช้งาน ควรปรับเทียบด้วยข้อมูลจริงก่อนใช้เชิงผลิตภัณฑ์หรือเชิงคลินิก

## ข้อมูลที่เก็บ

ในโหมด browser-only ระบบเก็บ:

- โปรไฟล์ผู้รับการทดสอบ
- metadata ของ session
- metric รายเป้าหมาย
- จำนวนการเคลื่อนไหวในแต่ละช่องสำหรับ heatmap
- ข้อความรายงานอัตโนมัติ

การบันทึกวิดีโอเป็นตัวเลือกที่ต้องได้รับความยินยอมก่อน ในต้นแบบนี้วิดีโออยู่เป็น object URL ภายใน browser ระหว่างดู session และไม่มีการอัปโหลด

## วิธีรัน

```bash
pnpm install
pnpm dev
```

Build:

```bash
pnpm build
```

ค่าเริ่มต้นของ Vite dev server:

```text
http://127.0.0.1:5173/
```

## วิธีสาธิต

- ใช้โหมดผู้รับการทดสอบสำหรับ consent, profile, calibration และ reach task
- ใช้โหมดแพทย์/นักกายภาพเพื่อดูคะแนน heatmap รายงาน และส่งออก JSON
- หากกล้อง โมเดล หรือ permission ไม่พร้อม ให้ใช้โหมดสาธิตหรือสัญญาณสาธิตเพื่อดำเนิน flow ต่อ

## งานต่อไป

- ใช้ IndexedDB สำหรับ artifact ขนาดใหญ่
- บันทึก WebM แบบถาวรเมื่อได้รับความยินยอม
- เพิ่ม FastAPI backend พร้อม SQLite และ local file storage
- เพิ่ม API สำหรับ session ของแพทย์และ workflow การส่งออกข้อมูล
- เพิ่ม assistant อ้างอิง clinical knowledge ด้วย LangChain/FAISS
- เตรียม object storage, audit log และ retention policy สำหรับระบบจริง
